C++ is the most misunderstood language in the embedded world. People who learned it from a 2003 textbook remember `iostream`, multiple inheritance, RTTI, exceptions, and a fat runtime — and they refuse to touch it. People who learned it from a 2019 talk remember `constexpr`, ranges, concepts, and zero-overhead abstractions — and they cannot imagine why anyone would still write C. The truth lives between the two, and the difference comes down to which subset of the language you actually use.

This tutorial teaches the modern subset. The features that earn their keep: classes and RAII, templates, the STL containers you will actually reach for, smart pointers, move semantics, lambdas, ranges, and the C++23 additions like `std::expected`. It skips the historical baggage you can safely ignore: `iostream` formatting machinery, RTTI, exceptions on small targets, multiple inheritance soup, and the SFINAE tricks that concepts replaced.

By the end you will be able to read modern C++ codebases, write small libraries, use the STL without surprises, and — in the final lesson — apply what you learned to a Cortex-M target with `arm-none-eabi-g++`, knowing exactly which features cost what.

## Philosophy

Three rules govern every lesson.

**Use the features that pay for themselves at compile time.** Templates, `constexpr`, `concepts`, and the type system are free at runtime — they cost compile time and they buy you safety. Use them liberally.

**Be skeptical of features that pay at runtime.** Virtual functions, `std::function`, `std::shared_ptr`, exceptions, and dynamic allocation all have costs that show up in your final binary and your worst-case execution time. Use them when you need polymorphism, ownership counting, or error propagation that the alternatives cannot give you — not because the textbook said you should.

**Embedded relevance is a callout, not a detour.** Every lesson has a "Embedded callout" section that says what changes when you target a Cortex-M with no heap, no exceptions, and a 100 KB flash budget. Most C++ features survive that constraint with minor tweaks. A few do not. You will know which by lesson 8.

## How this differs from the reference page

[`/docs/programming/cpp/`](/docs/programming/cpp/) is the **reference**: a flat list of C++ features with embedded-flavoured advice on each, suitable for someone who already knows C++ and wants to know what to use on Cortex-M. It does not teach the language; it filters it for firmware engineers.

This tutorial is the **course**: an ordered 15-lesson path from "I know C, what does C++ add?" to "I can write a templated, RAII-managed, move-aware library and ship it on bare metal." Read this first. After lesson 15, the reference page becomes a quick lookup for any feature you forgot. The two are paired: tutorial for learning, reference for living.

## How to use this tutorial

Four rules.

1. **Read in order.** Each lesson assumes the last. Lesson 4 (inheritance) makes no sense without lesson 3 (classes). Lesson 7 (algorithms) assumes the containers from lesson 6. The dependencies are real and they are dense.

2. **Type the code.** Every code block in this tutorial is real, compileable, and short. Type it into a file, compile with `g++ -std=c++23 -Wall -Wextra -Wpedantic`, and run it. Reading C++ from a webpage and never compiling is the most common way to fool yourself into thinking you have learned C++.

3. **Do the exercises before reading the answers.** Two or three exercises end each lesson. Answers are in `<Callout type="spec">` blocks below them. Skipping the exercises and reading the answers feels efficient and is the slowest way to learn.

4. **Use the warnings.** `-Wall -Wextra -Wpedantic` catches real bugs. When the compiler complains, read the message. C++ compiler errors look ugly but they are usually right. The first 20 you decode by hand will earn you a sense for the next 200.

<Callout type="tip" title="Pick one editor with good C++ tooling">
clangd (via VS Code, Neovim, or Emacs), or CLion, or Visual Studio with its C++ tooling. Modern C++ is enormously easier to learn with go-to-definition and instant error squiggles than without them. Spend 30 minutes setting up clangd before lesson 1 and the rest of the tutorial will be faster.
</Callout>

## What you will not learn here

This tutorial deliberately omits:

- **Multiple inheritance and virtual inheritance** beyond a one-paragraph mention. They exist; they are rarely the right answer; the patterns that justify them (mixins, interface inheritance with default implementations) are better served by composition plus templates.
- **The full template metaprogramming toolkit.** SFINAE, `std::enable_if`, tag dispatch — concepts (C++20) replaced 90% of this. We mention the old names so you can read existing code, and move on.
- **C++ exceptions as the primary error model.** We cover them in lesson 11 alongside the alternatives (`std::optional`, `std::expected`). For embedded targets, exceptions are usually off.
- **Boost and other large external libraries.** We stick to the standard library so the lessons compile anywhere `g++` runs.

If you need any of these later, the C++ Programming Language reference (Stroustrup) and cppreference.com are exhaustive. This tutorial gets you to the point where those references are useful.

## Compiler setup, in 30 seconds

On macOS:

```
brew install gcc
g++-14 --version
```

On Linux:

```
sudo apt install g++-13     # Ubuntu 24.04 has g++-13; 24.10 has g++-14
g++ --version
```

On Windows: install [MSYS2](https://www.msys2.org/) and `pacman -S mingw-w64-ucrt-x86_64-gcc`, or use WSL.

Every example in this tutorial compiles with:

```
g++ -std=c++23 -Wall -Wextra -Wpedantic -O2 file.cpp -o file
```

If your compiler does not support `-std=c++23`, fall back to `-std=c++20`. Lessons 13 and 14 will then need adjustments noted inline. `clang++` works identically — replace `g++` with `clang++` in every command.
